# Руководство по обеспечению безопасности (ФСТЭК)
## МИАЦ.СВЯЗЬ — Панель управления Asterisk

Данный документ описывает меры по обеспечению информационной безопасности системы МИАЦ.СВЯЗЬ в соответствии с требованиями регуляторов для систем, работающих на базе **AltLinux SP 10**.

### 1. Архитектура закрытого контура (Air-Gap Ready)
Система спроектирована для работы в изолированных сетях без доступа к интернету:
- **Отсутствие внешних зависимостей**: Удалены все обращения к внешним CDN, включая Google Fonts и сторонние библиотеки скриптов.
- **Локальные шрифты**: Используются стандартные системные шрифты ОС.
- **Локальное хранение**: Все конфигурационные данные и секреты (пароли SIP) хранятся в защищенном каталоге `/etc/asterisk/` и локальных JSON-файлах.

### 2. Идентификация и Аутентификация
- **Локальная база администраторов**: Доступ к панели осуществляется через локальную базу данных (`src/data/admins.json`).
- **Хеширование паролей**: Пароли хранятся исключительно в виде хеш-сумм SHA-256 с использованием соли.
- **Управление сессиями**: Используются HTTP-only куки с ограниченным временем жизни для предотвращения атак типа Session Hijacking.

### 3. Разграничение доступа (RBAC)
Система реализует ролевую модель доступа:
- **Администратор**: Полный доступ к конфигурации транков, маршрутов, IVR и управлению пользователями.
- **Оператор (в разработке)**: Доступ только к мониторингу статусов и просмотру логов.

### 4. Защита коммуникаций и данных
- **Интеграция с Asterisk**: Взаимодействие с Asterisk происходит через локальный интерфейс AMI (127.0.0.1).
- **Секреты PJSIP**: Генерируемые файлы конфигурации (`pjsip_miac_users.conf`) содержат пароли абонентов. Доступ к этим файлам ограничен правами файловой системы ОС (666/644).
- **Безопасность AMI**: Для управления используется выделенный пользователь `miac` с ограниченными правами доступа только к необходимым командам.

### 5. Регистрация событий (Аудит)
- **Системный лог**: Все действия «Моста» и ошибки синхронизации фиксируются в реальном времени и доступны для просмотра в разделе «Управление».
- **Логи Asterisk**: Все изменения конфигурации логируются стандартными средствами Asterisk в `/var/log/asterisk/messages`.

### 6. Целостность системы
- **Контроль конфигурации**: Генерируемый диалплан проходит проверку на корректность перед перезагрузкой Asterisk, что исключает возможность отказа в обслуживании из-за синтаксических ошибок.
- **Валидация ввода**: Все вводимые данные (номера, шаблоны) проходят строгую валидацию через схемы Zod.
